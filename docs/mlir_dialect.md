# `qfx.timeseries` MLIR Dialect Specification

The `qfx.timeseries` (or simply `qfx`) dialect is the core intermediate representation of the `quantfx-compiler`. It models high-level financial time-series operations, allowing the compiler to perform domain-specific optimizations (such as window fusion and streaming incremental updates) before lowering to standard generic loops.

## Motivation

Standard LLVM IR or MLIR's `linalg`/`affine` dialects lack the semantic knowledge of "sliding windows" and "tick data streams". Without this knowledge, operations like a rolling VWAP and a rolling volatility over the same window might be compiled as two separate heavy loops over memory.

By preserving the semantics of operations like `qfx.rolling_vwap` in the IR, our custom passes can fuse them into a single loop, vastly reducing memory bandwidth.

## Dialect Types

Currently, `qfx` operates directly on MLIR's built-in `MemRef` types (e.g., `memref<?xf32>`), representing one-dimensional time-series streams.

## Core Operations

### `qfx.rolling_mean`
Computes a centered or trailing rolling mean over a stream.
- **Inputs:** `input` (MemRef), `window` (Integer attribute).
- **Output:** `output` (MemRef).
- **Semantics:** Similar to NumPy's `convolve(x, ones(W)/W, mode='same')` when not in streaming mode.

### `qfx.rolling_std`
Computes the rolling standard deviation.
- **Inputs:** `input` (MemRef), `window` (I64Attr).
- **Output:** `output` (MemRef).

### `qfx.rolling_vwap`
Computes the Volume-Weighted Average Price.
- **Inputs:** `prices` (MemRef), `volumes` (MemRef), `window` (I64Attr).
- **Output:** `output` (MemRef).

### `qfx.zscore`
Computes the element-wise z-score given a stream, its rolling mean, and rolling standard deviation.
- **Inputs:** `input`, `mu`, `sigma` (all MemRefs).
- **Output:** `output` (MemRef).

### `qfx.fused_rolling_stats`
A fused operation generated by the **Window Fusion Pass** when a `qfx.rolling_mean` and `qfx.rolling_std` share the same input and window size. It computes both in a single pass to save memory bandwidth.
- **Outputs:** `mean`, `stddev` (MemRefs).

## Lowering Strategy

The compilation pipeline lowers the `qfx` dialect progressively:

1. **qfx → affine / linalg**: High-level rolling operations are expanded into perfect loop nests using the `affine.for` loop construct. The window boundaries are mapped to loop bounds.
2. **affine → vector**: Loop tiling and vectorization passes convert inner `affine` loops over floating-point values into `vector` dialect operations (e.g., `vector.fma`, `vector.broadcast`).
3. **vector → llvm**: The vector operations map directly to LLVM IR intrinsics (like AVX-512 instructions on CPU, or PTX vector instructions on GPU).

### Streaming Semantics

If the `--streaming` flag is passed, the `qfx` lowering pass converts operations into stateful accumulators (e.g., Welford's algorithm for rolling variance) rather than fully recomputing the window for every tick.
